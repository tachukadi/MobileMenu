## Technical Requirements

### Image Specifications
- Format: PNG or JPG
- Dimensions: Minimum 1200px width (1280x800 recommended)
- Aspect Ratio: 4:3 or 16:9
- File Size: Under 1MB each
- Quality: High resolution, clear text

### Naming Convention
```
screenshot-1.png (or .jpg)
screenshot-2.png
screenshot-3.png
... up to screenshot-8.png
```

### Where to Place
Place all screenshots in the plugin's `/assets/` directory when submitting to WordPress.org

## Creating Screenshots

### Tools You Can Use
1. **Browser DevTools**
   - Open DevTools (F12)
   - Toggle device toolbar
   - Select mobile device
   - Take screenshot

2. **Mac**
   - Cmd + Shift + 4 for selection
   - Cmd + Shift + 3 for full screen

3. **Windows**
   - Win + Shift + S for Snipping Tool
   - PrtScn for full screen

4. **Screenshot Tools**
   - Awesome Screenshot (browser extension)
   - Snagit
   - Lightshot
   - Skitch

### Best Practices
- Use real content, not Lorem Ipsum
- Show the plugin actually working
- Use professional color scheme
- Ensure text is readable
- No personal information visible
- Consistent theme/design across screenshots
- Good lighting/contrast
- Clear, focused shots

## Banner Images (Optional)

For better WordPress.org presentation:

### Plugin Header Banner
- Filename: `banner-772x250.png`
- Dimensions: 772 x 250 pixels
- Shows: Plugin name, tagline, key features

### High-Resolution Banner
- Filename: `banner-1544x500.png`
- Dimensions: 1544 x 500 pixels
- Shows: Same as above, retina version

### Plugin Icon
- Filename: `icon-128x128.png` and `icon-256x256.png`
- Dimensions: 128x128 and 256x256 pixels
- Shows: Plugin logo/symbol

## Example Screenshot Workflow

1. Install plugin on a clean WordPress site
2. Create a sample menu with various items
3. Configure the plugin with attractive settings
4. Open browser DevTools, set to iPhone/iPad view
5. Take screenshot of closed state
6. Open menu, take screenshot
7. Expand submenu, take screenshot
8. Switch to admin, capture each settings tab
9. Optimize images for web
10. Name files correctly
11. Place in `/assets/` directory

**Note**: These screenshots are crucial for WordPress.org approval and user understanding. Take time to make them professional and clear.
